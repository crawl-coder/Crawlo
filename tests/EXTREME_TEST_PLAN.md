# Crawlo 框架极限场景测试方案

## 测试目标
从用户角度验证框架在**常规场景**、**边界条件**、**异常情况**下的健壮性和可靠性。

---

## 一、网络请求层测试 (Network)

### 1.1 常规场景
- [ ] 正常 HTTP GET/POST 请求
- [ ] 带 Headers、Cookies、Proxy 的请求
- [ ] JSON/表单数据提交
- [ ] 文件上传下载
- [ ] 分页请求（自动翻页）

### 1.2 边界场景
- [ ] **空 URL** → 应抛出清晰错误
- [ ] **超长 URL**（10000+ 字符）→ 应正常处理或限制
- [ ] **特殊字符 URL**（中文、空格、emoji）→ 应正确编码
- [ ] **非法协议**（ftp://、file://）→ 应拒绝或提示
- [ ] **超时设置**（0s、负数、极大值）→ 应合理处理

### 1.3 异常场景
- [ ] **DNS 解析失败** → 应有重试机制
- [ ] **连接拒绝** → 应记录错误并继续
- [ ] **SSL 证书错误** → 可配置忽略或拒绝
- [ ] **429 Too Many Requests** → 应自动降速
- [ ] **500/502/503 服务器错误** → 应重试
- [ ] **响应体超大**（100MB+）→ 应截断或拒绝
- [ ] **响应编码错误** → 应 fallback 到 latin1

### 1.4 并发测试
- [ ] **100 并发请求** → 不崩溃，无连接泄漏
- [ ] **1000 并发请求** → 内存不爆炸
- [ ] **快速创建/销毁请求对象** → 无资源泄漏

---

## 二、队列系统测试 (Queue)

### 2.1 内存队列
- [ ] **正常入队/出队** → FIFO/优先级正确
- [ ] **空队列 get()** → 返回 None 或超时
- [ ] **队列满** → 背压机制生效
- [ ] **10000 个请求入队** → 性能不下降

### 2.2 Redis 队列
- [ ] **Redis 未启动** → 优雅降级或清晰错误
- [ ] **Redis 连接断开** → 自动重连
- [ ] **Redis 内存满** → 错误处理
- [ ] **脏数据存在** → 自动清理
- [ ] **集群模式** → hash_tag 正确

### 2.3 批量操作
- [ ] **put_batch 空列表** → 返回 0
- [ ] **put_batch 1000 个请求** → 性能提升
- [ ] **put_batch 部分失败** → 返回成功数
- [ ] **put_batch 超大 batch_size** → 自动分批

### 2.4 优先级测试
- [ ] **优先级反转** → 小值先出队
- [ ] **相同优先级** → FIFO
- [ ] **优先级动态调整** → 正确排序

### 2.5 背压测试
- [ ] **队列满时继续入队** → 阻塞或拒绝
- [ ] **背压阈值触发** → 日志警告
- [ ] **背压解除** → 自动恢复

---

## 三、爬虫引擎测试 (Engine)

### 3.1 生命周期
- [ ] **正常启动→运行→关闭** → 无资源泄漏
- [ ] **立即关闭**（启动后 0.1s）→ 优雅退出
- [ ] **多次启动** → 应拒绝或警告
- [ ] **关闭后再次操作** → 应提示已关闭

### 3.2 异常处理
- [ ] **Spider 抛出异常** → 不影响其他 Spider
- [ ] **Middleware 抛出异常** → 有 fallback
- [ ] **Pipeline 抛出异常** → 数据不丢失
- [ ] **无限循环 start_requests** → 有超时保护

### 3.3 并发控制
- [ ] **并发数=1** → 串行执行
- [ ] **并发数=100** → 不崩溃
- [ ] **并发数=负数** → 使用默认值
- [ ] **动态调整并发数** → 平滑过渡

### 3.4 资源管理
- [ ] **内存监控** → 不泄漏
- [ ] **文件句柄** → 及时关闭
- [ ] **数据库连接** → 连接池回收
- [ ] **Redis 连接** → 优雅关闭

## 测试报告格式

每个测试用例应包含：
1. **测试名称** - 清晰描述测试内容
2. **前置条件** - 测试环境要求
3. **测试步骤** - 详细操作步骤
4. **预期结果** - 期望的行为
5. **实际结果** - 实际的行为
6. **测试状态** - PASS/FAIL/SKIP
7. **错误信息** - 失败时的详细错误
8. **测试时间** - 执行耗时
9. **备注** - 其他说明
