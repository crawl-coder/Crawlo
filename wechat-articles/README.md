# Crawlo 微信公众号推广方案

## 一、公众号定位

| 维度 | 定义 |
|------|------|
| **账号名称** | Crawlo 爬虫框架（暂定） |
| **定位** | Python 爬虫技术分享 + Crawlo 框架推广 |
| **目标读者** | Python 开发者、数据工程师、爬虫工程师、爬虫初学者 |
| **内容调性** | 技术硬核 + 通俗易懂，避免纯广告感 |

## 二、核心卖点提炼

从框架设计中提炼 5 大差异化卖点，所有文章围绕这些卖点展开：

| # | 卖点 | 一句话描述 | 对标 Scrapy 痛点 |
|---|------|-----------|-----------------|
| 1 | **asyncio 原生** | Python 原生异步，不用学 Twisted | Scrapy 绑死 Twisted，回调地狱 |
| 2 | **断点续爬** | 崩溃自动保存，重启接着爬 | Scrapy Jobdir 体验差，不支持数据状态 |
| 3 | **智能背压** | 自动限速，不怕被封号 | Scrapy 无背压，手动调参 |
| 4 | **Cloudflare 绕过** | 内置反检测，一行配置搞定 | Scrapy 需装多个第三方中间件 |
| 5 | **分布式就绪** | 单机→多机→生产级，改一行配置 | Scrapy-Redis 停更多年，无 ACK 语义 |
| 6 | **定时调度** | Cron 表达式 + 间隔调度，设好自运行 | 需外部 crontab 管理 |

## 三、文章系列规划

### 第一阶段：认知建立（3 篇，每周 1 篇）

|------|------|---------|------|

| 2 | **3 分钟写一个爬虫：Crawlo 极速上手指南** | 安装→写 Spider→运行→存储，完整流程 | 让读者能跑起来 |

| 5 | **被封号？被限速？Crawlo 智能背压让你安稳爬取** | 背压策略 + 自适应算法 + 效果对比 | 展示安全性 |
